Enumerable Methods

The map method

July 12, 2015

This week at Dev Bootcamp we learned a ton about enumerable methods. Enumerable methods are useful because they give you a lot of ways of doing something to every element of a collection of objects (array, hash, etc.). In this blog post I will be explaining how to utilize the enumerable map method. I chose to elaborate on this method because I found myself using it a lot and it also doubles for an explanation of the enumerable method collect.

To give a brief summation, #map returns a new array filled with whatever gets returned by the block each times it runs. In ruby #map and #collect do exactly the same thing, however it is probably considered better ruby convention to use the #map method. To think of it in the simplest terms the map method is a re-mapping of your original objects through a transformation that occurs when you use #map. Map can be called in its' destructive (!) or non destructive form. If it called using map!, the objects will be transformed permanently.
