Array's vs Hashes

June 29, 2015

This week at Dev Bootcamp we touched on the Ruby basics. I am finding Ruby to be an incredibly fun language to learn but challenging all at the same time. I have definitely experienced more than a few "ah-ha" moments this week. A few of those moments came when exploring the world of arrays and hashes. In this post I am going to go over the main difference between the two and how to utilize each.

Array

In Ruby everything is classified as an object. Arrays are ordered collections of an object. You can think of an array as a list that can contain any kind of object (string, integer, float) so long as the data is in some kind of order. The order in an array is important because you access the data by using the number it corresponds to on the list. For example, let's say you made a shopping list with milk as the first item, bread as the second, and cheese as the third. Now, let's say you wanted to access each item in the list: 

shopping_list = ["milk", "bread", "cheese"]

shopping_list[0] => "milk"

shopping_list[1] => "bread"

shopping_list[2] => "cheese"

Hashes

Hashes, like arrays, are collections of data except with hashes the data is stored as pairs of information. Each pair has a name and a value. The name that is used to identify a hash is called a key and it is unique to the pair. The key is what allows you to access the value that is paired to it. Let's use the shopping list example again. Let's say you wanted to get really specific with your grocery list. Instead of bread you want sourdough and instead of milk you want almond milk. To make a specified list you could use a hash. 

shopping_list { "milk" => "almond", "bread" => "sourdough", "cheese" => "gouda"}

shopping_list ["milk"] => "almond"

shopping_list ["bread"] => "sourdough"

shopping_list ["cheese"] => "gouda"
